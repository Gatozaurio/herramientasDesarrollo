Cada Alumno debe hacer un trabajo sobre las funcionalidades que ofrece las herramientas de desarrollo de un navegador. El trabajo deberá presentarse como una página una página web con una lista de enlaces a otras páginas Web, dónde se explique la funcionalidad, con texto, vídeo o imágenes. Además el proyecto deberá cumplir los siguientes requisitos:

  • Debe explicarse cada funcionalidad que ofrece las herramientas
  • Debe existir una página principal dónde se accede a todas las funcionalidades detalladas.
  • Debe realizarse un ejemplo, mediante captura de pantallas o vídeos de cada una de las funcionalidades.
  • Se evaluará con un 0.5 cada funcionalidad explicada correctamente, junto a sus capturas o videos. El mínimo son 5 puntos.
  • Las funcionalidades obligatorias a explicar y detallar mediante ejemplos son(Si no se hace alguna de estas funcionalidades el trabajo estará suspenso):
    • Debugger: Añadir breakpoint y explicar como ir pasa a paso, terminar ejecución o entrar dentro de un método.
    • Debugger: Ver el valor de expresiones y variables en un momento concreto en la ejecución de un código.
    • Mostrar dónde aparecen todos los recursos que se ha descargardo la web para su funcionamiento.
    • Ver el código fuente de la página y modificarlo(incluyendo un nuevo elemento, por ejemplo).
    • Ver el css asociado a un elemento de la página, indicar las reglas css que afectan a dicho elemento y modificar el estilo css online.
    • Mostrar todos los tipos de errores que podemos ver en consola y ejecutar algunas operaciones en la consola.
    • Detallar la información de un recurso obtenido desde la página web:
      • Tiempo de carga
      • Cabeceras de la petición: Detallar que significa cada uno de los apartados de este caso, e indicar algún ejemplo. (Esto se considera una funcionalidad aparte)
      • Preview
      • Respuesta del servidor
    • Filtrar por tipo los recursos que obtenemos e indicar que significa cada tipo.
  • Debe enviarse el enlace del repositorio de git al profesor para que pueda descargarse el trabajo.
  • Cada semana debe incluirse al menos una nueva funcionalidad al trabajo. Si no se incluye, recibirá un strike, al cuarto strike,el alumno deberá presentarse a esta parte en el examen.
